# GitHub Copilot Chat 入门教程

（此文件将在提交到仓库后另行转换为 PDF；当前为 Markdown 源，含示意图描述、示例对话与练习题）

## 1 快速概览

- 什么是 GitHub Copilot Chat：在 GitHub.com 上集成的智能聊天助手，能帮助你写代码、解释代码、生成测试、创建 PR 描述、排错等（相当于针对代码库/任务的交互式 AI 助手）。
- 适合谁：开发者、新手、code reviewer、文档维护者等。对初学者尤其友好，因为可以“对话式”学习与调试。
- 注意事项：是否可用取决于你的 GitHub 账户/组织许可（部分功能需要 Copilot 订阅或企业启用）。请联系组织管理员或查看你的账号设置如果看不到功能。

## 2 前置条件（准备工作）

- 一个 GitHub 账户（github.com 注册）。
- 建议：在使用前登录到 GitHub（在同一浏览器会话里）。
- 可能需要的订阅/权限：Copilot Chat 可能需要 Copilot 订阅或由组织管理员开启“Copilot for Business”相关权限。若你是学生/个人，请检查 Copilot subscription 页面或你的账户设置。
- 浏览器：建议使用 Chrome / Edge / Firefox 的最新版以获得最佳体验。

## 3 如何打开 Copilot Chat（网页版）

（说明：不同帐号/组织界面可能略有差异，下面给通用步骤）

- 方法 A（有 Copilot 图标时）：
  1. 登录 github.com。
  2. 在页面右上角或左侧导航寻找 “Copilot” 或 “Copilot Chat” 的图标/入口，点击打开聊天面板。
- 方法 B（从仓库或 PR 内）：
  1. 打开任意仓库页面或 PR 页面。
  2. 寻找页面上方或右侧的“Copilot”按钮（或“Chat”）。
  3. 点击后会在侧边或弹窗中打开聊天界面。
- 方法 C（直接 URL / 功能页）：
  - 若 GitHub 提供了产品页，可尝试访问 GitHub 的 Copilot功能页面，那里会有“打开 Copilot Chat”的入口。

## 4 界面导览（左、中、右三栏常见布局）

- 左侧（会话列表）：展示历史会话、已保存的会话或模板。
- 中间（聊天窗口）：与 Copilot 的对话历史、消息气泡、代码块、可折叠片段。
- 底部（输入框 / 工具栏）：输入 prompt、上传文件/粘贴代码、选择输出格式（有时可选择“只返回代码”/“以步骤说明”）。
- 右侧（上下文 / 工具）：可能显示与当前仓库相关的上下文（文件、最近的 PR、CI 报告）以及快捷动作（生成 PR 描述、创建 issue 等）。

## 5 第一次对话：从简单命令开始（逐步示例）

- 基本用法：在输入框直接写自然语言问题或任务，例如：
  - “请帮我写一个能把数组去重的 JavaScript 函数，并加上注释和 3 个测试用例。”
  - “我在 foo.js 的第 30 行遇到错误：ReferenceError: x is not defined，可能的原因有哪些？怎么修复？”
- 交互步骤：
  1. 在输入框写上你的需求（尽量明确，指出语言/风格/行数限制等）。
  2. 等待 Copilot Chat 生成回答。回答里通常包含解释、代码块、以及后续建议。
  3. 你可以继续追问、要求更改、要求输出不同格式（如 JSON、Markdown、patch 等）。

### 示例：生成去重函数

```javascript
// 去重示例：返回新数组，保留原始顺序
function dedupe(array) {
  return Array.from(new Set(array));
}

// 测试
console.log(dedupe([1,2,2,3])); // [1,2,3]
```

Prompt 示例： “请写一个 JavaScript 的 dedupe 函数，保留数组原始顺序，给出 3 个不同类型的测试用例，并解释复杂度。”

## 6 常用指令与 Prompt 模板（便于复制）

- 指令/模板 A（写代码）：
  - “用 <语言> 写一个 <功能>，要点：1) 简洁 2) 带注释 3) 带 3 个单元测试。代码用 markdown 的代码块返回。”
- 指令/模板 B（生成 PR 描述）：
  - “下面是本次提交的改动摘要（或粘贴 diff），请根据改动生成一段适用于 PR 的标题和 3 段式描述（原因、实现、测试）。格式：Markdown。”
- 指令/模板 C（代码审查）：
  - “请审查下面的代码并给出：1) Bug 风险点 2) 可改进的性能/可读性点 3) 一个简短的 refactor 建议。”
- 指令/模板 D（限制输出形式）：
  - “请只返回 JSON，格式为 {summary: '', steps: ['',''], files: [{'path':'','content':''}]}。不要额外说明。”

## 7 三个完整示例场景（实战）

- 场景 A：把函数转换为带类型注解的 TypeScript 版本
  - 你输入： “将下面的 JS 函数改为 TypeScript，并添加类型注释及接口说明：<代码>”
  - Copilot 输出：提供 .ts 代码块、类型定义、简单示例调用。
- 场景 B：自动生成单元测试
  - 你输入： “为下面的函数生成 Jest 单元测试，包含正常路径和边界条件：<代码>”
  - Copilot 输出：jest 测试文件内容与运行命令示例。
- 场景 C：从 commit diff 生成 PR 描述（工作流）
  - 步骤：
    1. 在 GitHub PR 页面打开 Copilot Chat（或把 diff 复制粘贴到对话）。
    2. Prompt： “基于下面的 diff，写一个 PR 描述：一句话的标题 + 3-4 段说明（目的、变更概述、测试方法、回归考虑）。”
    3. Copilot 返回结构化 PR 描述，你可以直接复制到 PR 描述框中。

## 8 隐私、数据使用与设置（重要）

- 数据去向（一般说明）：
  - Copilot Chat 需要把你的提示和（可选）仓库上下文发送到后端模型进行处理。不同组织可配置是否允许发送内部仓库上下文或启用数据保留策略。
- 控制项（你可以检查/设置的地方）：
  - 账户/组织安全设置：查看组织是否启用了 Copilot 功能与数据共享策略。
  - Copilot 设置面板：通常可设置对话保存、是否将对话用于模型改进（opt-in/opt-out）、是否自动包含仓库文件上下文等。
- 建议：
  - 不要在 prompt 中包含明文密码、密钥或敏感凭据。若需要在对话中包含敏感信息，先了解组织的合规策略或使用脱敏示例替代真实秘密。
  - 若对隐私极度敏感，向组织管理员咨询“不上传代码”或“仅本地运行”方案（有时有私有部署/本地化选项）。

## 9 常见问题与排查（FAQ）

- Q：我看不到 Copilot Chat 的入口。怎么办？
  - A：确认你已登录、检查账号是否具有 Copilot 订阅或组织是否启用该功能，或尝试清理浏览器缓存 / 使用另一个浏览器 / 私有窗口登录。
- Q：Copilot 给出的代码不工作或含错误？
  - A：把错误信息与上下文贴到对话里，要求 Copilot debug 并给出修复步骤；永远把 AI 输出当作“建议”，并在合适的测试与 code review 后合并。
- Q：如何把 Copilot 输出整合到 CI？
  - A：可以让它生成 patch（diff）或完整文件内容，然后在本地/CI 中验证并提交；不要直接自动合并没有人工审核的 AI 输出。
- Q：能否导出对话或保存模板？
  - A：许多实现支持导出会话、保存为模板或收藏常用 prompt。查看左侧会话列表的保存/导出选项。

## 10 进阶技巧（提高效率的做法）

- 精确指定输出格式（JSON / Markdown / 代码块），方便自动化解析。
- 要求分步说明（“先列出思路，再给代码”）可以减少误解。
- 给出示例输入与预期输���以创建“合同式 prompt”（contract prompt）。
- 使用“上下文上传”功能（若可用）把相关文件或 diff 上传，以获得更准确的建议。
- 在生成复杂改动前，先让 Copilot 提供“变更计划（步骤）”，审阅计划后再生成代码。

## 12 练习题与答案（含解析）

### 练习 1：生成去重函数

题目：使用 JavaScript 写一个名为 dedupe 的函数，要求：保留原数组顺序、处理非数组输入（返回空数组）、并写 3 个测试用例。

参考答案：

```javascript
function dedupe(arr) {
  if (!Array.isArray(arr)) return [];
  const seen = new Set();
  const res = [];
  for (const item of arr) {
    if (!seen.has(item)) {
      seen.add(item);
      res.push(item);
    }
  }
  return res;
}

// 测试
console.log(dedupe([1,2,2,3])); // [1,2,3]
console.log(dedupe(['a','b','a'])); // ['a','b']
console.log(dedupe(null)); // []
```

解析：使用 Set 记录已出现元素，遍历时保持顺序，先判断输入类型以提高鲁棒性。

### 练习 2：生成 PR 描述

题目：假设你在一个仓库中改动了 README 和 utils/parse.js，请写出一个 PR 的标题和 3 段式描述（目的、实现、测试）。

参考答案：

标题：Improve README and fix parsing edge-case in parse.js

描述：
1) 目的：清晰说明 README 中示例的运行步骤，并修复 parse.js 在处理空字符串时崩溃的问题。
2) 实现：更新 README 的示例命令、增加介绍节；在 utils/parse.js 中加入空字符串判断并添加相关单元测试。
3) 测试：已在本地运行单元测试 (npm test)，并手动验证 README 中的示例命令在新环境中可成功执行。
